## Installation

This repository requires **Python 3.11.14**.  
You can create the environment using **Conda** and install dependencies with **pip**.

<!-- omit from toc -->
### pip

1. Create a Python 3.11.14 environment:
   ```bash
   conda create -n pointing python=3.11.14
   ```

2. Activate the environment and install the required packages:
   ```bash
   conda activate pointing
   pip install -r requirements.txt
   ```

> [!IMPORTANT]
> For running the experiments with molmo, you will have to (optionally create a new environment and) install the additional packages specified in the [molmo_requirements.txt](./molmo_requirements.txt) file.



## Available Models

The following models can be specified using the `MODEL_NAME` argument in the commands below:

- qwen2.5-vl-3b
- qwen2.5-vl-7b
- llava-onevision-78
- internvl3-8b

### Adding a New Model
New models can be added by creating a collator in the [utils](./utils) folder. You can check an example by looking at:

- [utils.py](./utils/utils.py) `MAP_MODELS`, which contains the list of available models
- [qwen.py](./utils/qwen2_vl.py)



## Data

Data was generated by leveraging the [CIVET framework](https://aclanthology.org/2025.findings-emnlp.239/).

The datasets used in our experiments are located in the [`data`](./data) folder.
Before running experiments, unpack the required archives:

```bash
tar -xzvf ARCHIVE_NAME.tar.gz
```

> [!IMPORTANT]
> Due to its size, the `noisy_ts` dataset is attached to the release. Download the corresponding archive and place it into the data folder.

## Training

All examples in this section use [Qwen2.5-VL-3B-Instruct](https://huggingface.co/Qwen/Qwen2.5-VL-3B-Instruct).
You can substitute any other model listed in [Available Models](#available-models).

```bash
python -m main qwen2.5-vl-3b --out-dir your_results fine-tune data/DATA_NAME EXP_FOLDER
```

- `DATA_NAME` → name of the dataset folder (i.e., `base` or `noisy_tr`)
- `EXP_NAME` → name assigned to the experiment (results will be saved to `your_results/qwen2.5-vl-3b/EXP_NAME`). The model's weights will be located under the `best_model` folder.


> [!IMPORTANT]
> By default, the models are fine-tuned for Direct Counting (DC), you can instead train them for:
> - Point-then-Count (PtC), by using the optional argument `--add-pointing-coords`
> - X Fine-Tuning, by using the optional argument `--add-xs`
>
> Please, run the command with --help for the full list of arguments


## Evaluation

All examples in this section use [Qwen2.5-VL-3B-Instruct](https://huggingface.co/Qwen/Qwen2.5-VL-3B-Instruct).
You can substitute any other model listed in [Available Models](#available-models).

### Pre-Trained Model

Run the following command to evaluate a model:
```bash
python -m main qwen2.5-vl-3b --out-dir your_results evaluate data/DATA_NAME EXP_NAME
```

Where:
- `DATA_NAME` → name of the dataset folder (e.g., `id`, `ood`, or `noisy_ts/9_distractors`)
- `EXP_NAME` → name assigned to the experiment (results will be saved to `your_results/qwen2.5-vl-3b/EXP_NAME`)

### Fine-Tuned Models

Run the following command to evaluate a fine-tuned model:
```bash
python -m main qwen2.5-vl-3b --out-dir your_results evaluate data/DATA_NAME EXP_NAME --model-checkpoint PATH/TO/WEIGHTS
```

Where `PATH/TO/WEIGHTS` is the folder containing the `adapter_model.safetensors`.

> [!IMPORTANT]
> By default the checkpoint with the lowest perplexity will be saved under the `best_model` folder.


#### Checkpoints
Checkpoints for each model can be found in the `DC` and `PtC` directories under:
- `base-weights`, when trained on the [`base`](./data/base.tar.gz) dataset;
- `noisy_tr-weights`, when trained on the [`noisy_tr`](./data/noisy_tr.tar.gz) dataset.

For instance, to evaluate an existing model on the `ood` setting it is possible to run the following command:
```bash
python -m main qwen2.5-vl-3b --out-dir your_results evaluate data/ood ood --model-checkpoint output/qwen2.5-vl-3b/PtC/base-weights/best_model
```

## Ablations

We report the commands to re-run the ablations experiments on PtC models.


### Image
Substitute the input image with a black image by running the following command:
```bash
python -m main qwen2.5-vl-3b --out-dir your_results ablations data/DATA_NAME EXP_NAME --ablate-image
```

Where `PATH/TO/WEIGHTS` is the folder containing the `adapter_model.safetensors` of a **PtC model**.


Perform a leave-one-out ablation on the coordinates by running the following command:
```bash
python -m main qwen2.5-vl-3b --out-dir your_results ablations data/DATA_NAME EXP_NAME --ablate-image
```

Where `PATH/TO/WEIGHTS` is the folder containing the `adapter_model.safetensors` of a **PtC model**.
